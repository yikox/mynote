# 并行策略

> 百万 token 长序列训练的两条并行路线：DeepSpeed-Ulysses 与 Ring Attention。

---

## 概览

- **主题范围**：本目录聚焦"Transformer 注意力 O(n²) 增长"在百万 token 训练中的两条主流解决路线。
- **共同目标**：把单卡放不下的长序列分布到多卡，本质都是"切序列 + 通信换内存"。
- **核心差异**：
  - **Ulysses**（Snowflake）：在**序列维度**和**注意力头维度**同时切分，靠 All-to-All 通信重排数据
  - **Ring Attention**（Berkeley Flash Attention 团队）：环形拓扑 + 块式计算 + 通信计算流水线
- **学习路径**：可任选一篇读起，再对照另一篇——重点看"通信模式"的差异。

---

## 知识地图

### 序列并行（Snowflake 路线）
- 原理：SP × Attention Head Parallel，All-to-All 两次完成"序列分片 → 头分片 → 还原"
- 扩展：4x 序列长度（来自原文）；适合中等规模集群

- [DeepSpeed-Ulysses序列并行.md](./神经网络/并行策略/DeepSpeed-Ulysses序列并行.md)

### 环形注意力（Berkeley 路线）
- 原理：块式注意力 + 环形 KV 传递 + 计算通信流水线
- 扩展：理论上无限线性扩展；通信与计算几乎完全 overlap

- [Ring-Attention环形注意力.md](./神经网络/并行策略/Ring-Attention环形注意力.md)
