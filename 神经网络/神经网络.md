- **主题范围**：以 LLM 为核心的 AI 应用层与基础设施层——模型架构、Agent 上下文、AIGC 推理优化、长序列并行训练、GPU 加速图像处理。
- **学习路径**：建议先看 [LLM应用设计指南](./神经网络/LLM应用设计指南.md) 建立整体认知 → [大语言模型架构分析](./神经网络/大语言模型架构分析.md) 看具体架构 → 再按需深入子目录。

## 知识地图

### LLM 应用与架构
LLM 知识分两层：上层是"如何让 LLM 真正可用"（设计指南），下层是"模型本身怎么搭"（架构分析）。两者构成从应用到原理的纵向通路。

- [LLM应用设计指南.md](./神经网络/LLM应用设计指南.md) — LLM Wiki 三层架构（Schema / Wiki / Query），把"一次性问答"演进为"持久化知识库"。
- [大语言模型架构分析.md](./神经网络/大语言模型架构分析.md) — 四大开源 LLM（DeepSeek V4 / GLM-5.1 / MiniMax M2.7 / Kimi K2.6）的 MLA、MoE、PD 分离部署。

### Agent 工程化
Agent 时代的关键瓶颈是上下文管理而非上下文窗口大小。本目录讨论 2026 年生产级 Agent 在长任务中循环推理、累积状态时面临的工程问题。

- [Agent.md](./神经网络/Agent/Agent.md) — 上下文管理知识地图
- [上下文管理模块设计.md](./神经网络/Agent/上下文管理模块设计.md) — Agent 上下文管理模块设计

### AIGC 推理优化
围绕 NVIDIA GPU 的 AIGC 推理工具链：MPS 解决多进程 CUDA 协作，Polygraphy 解决 ONNX↔TensorRT 的模型检查与转换。

- [AIGC.md](./神经网络/AIGC/AIGC.md) — MPS / Polygraphy 工具链
- [Nvidia-MPS使用指南.md](./神经网络/AIGC/Nvidia-MPS使用指南.md) — 多进程 CUDA 服务
- [Polygraphy使用指南.md](./神经网络/AIGC/Polygraphy使用指南.md) — ONNX/TensorRT 模型检查与转换

### 长序列并行训练
百万 token 训练的两条主流路线：Ulysses（注意力头切分 + All-to-All）与 Ring Attention（环形拓扑 + 块式计算）。两者目标相同、实现思路不同，可对照阅读。

- [并行策略.md](./神经网络/并行策略/并行策略.md) — Ulysses / Ring Attention 对照
- [DeepSpeed-Ulysses序列并行.md](./神经网络/并行策略/DeepSpeed-Ulysses序列并行.md) — 注意力头并行 + All-to-All
- [Ring-Attention环形注意力.md](./神经网络/并行策略/Ring-Attention环形注意力.md) — 环形拓扑 + 块式计算

### GPU 加速图像处理
把传统图像处理任务（编解码）卸载到 GPU，吞吐量优先于延迟。

- [图像处理.md](./神经网络/图像处理/图像处理.md) — GPU 加速编解码
- [NvJPEG使用指南.md](./神经网络/图像处理/NvJPEG使用指南.md) — GPU 加速 JPEG 编解码
