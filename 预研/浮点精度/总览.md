## 1. 动机：为什么要一路降到 4-bit

深度学习的算力/显存带宽需求在过去 8 年膨胀了 3–4 个数量级，
模型参数/激活张量如果维持 FP32，存储与互连很快就会成为瓶颈。
精度的"压缩"是必然趋势，但**bit 不能无限砍**：

- 训练需要 **稳定的反向传播**：梯度数值跨度大、累加链长，
  任何舍入噪声都可能在矩阵乘法累积中被放大。
- 推理需要 **足够高的精度保留判别性**：尤其 KV cache 这种小而关键的张量。

压缩精度的两条主线：

1. **横向压位**：FP32 → FP16 → FP8 → FP4，单纯砍 bit 数。
   每砍一位就少一点动态范围或少一点精度，**没有外力补偿**。
2. **纵向加 scale**：给一小块元素共享一个 exponent，把"动态范围"找回来。
   OCP MX 与 NVIDIA NVFP4 都走这条路。

总览这一篇先讲通用结构（§2）和速查表（§3），再讲选型（§5）。

## 2. IEEE 754 通用结构

任意 IEEE 754 二进制浮点数可表示为

$$
x = (-1)^s \cdot 1.M_2 \cdot 2^{E - \text{bias}}
$$

其中

- **s**：1 bit 符号位
- **E**：e bit 指数位，存储的是 **带偏置值**（实际指数 = E − bias）
- **M**：m bit 尾数位，**隐含前导 1**，所以有效精度是 m + 1 bit
- bias 通常取 $2^{e-1} - 1$（IEEE 风格）

两个派生量决定格式的"形状"：

| 派生量 | 由谁决定 | 影响 |
|---|---|---|
| 动态范围 | 指数位宽度 $e$ | 表示数值的最大最小跨度，约 $2^{2^{e-1}}$ 量级 |
| 精度 ULP | 尾数位宽度 $m$ | 相邻两浮点数的相对间距，约 $2^{-m}$ |

→ 想扩范围就加指数位；想加精度就加尾数位；总位固定时**此消彼长**。

### 2.1 规格化、次规格化与特殊值

- **规格化数**（normal）：$E \in (0, 2^e - 2)$，隐含前导 1
- **次规格化数**（subnormal）：$E = 0$，隐含前导 0，用于平滑过渡到 0
- **零**：$E = 0, M = 0$，有 +0 与 −0
- **无穷**（±Inf）：$E = 2^e - 1, M = 0$
- **NaN**：$E = 2^e - 1, M \neq 0$

> FP8 的 E4M3 不编码 ±Inf，把那一组 bit 模式让出来给"额外的正常数"，
> 以换取更宽的精度——这是为深度学习刻意做的偏差。

### 2.2 舍入误差上界（向后稳定性的基础）

IEEE 754 默认 **round-to-nearest-even**（RTNE），
可证任意实数 $x$（在格式范围内）经一次量化后变为 $\hat{x}$，满足

$$
|\hat{x} - x| \le \tfrac{1}{2} \text{ulp}(x)
$$

其中 $\text{ulp}(x) \le 2^{-m} |x|$（规格化数）。
写成**相对误差**形式：

$$
\frac{|\hat{x} - x|}{|x|} \le 2^{-(m+1)} \quad (\text{规格化、RTNE})
$$

这一界称为 **machine epsilon** $\varepsilon = 2^{-m}$ 的同阶量。
对 FP32 这是 $\sim 5.96 \times 10^{-8}$；对 FP16 是 $\sim 4.88 \times 10^{-4}$。

> 这就是为什么长累加链（如矩阵乘的内积和、SoftMax 求和、Adam 的二阶矩估计）
> 在 FP16 下会出现"灾难性抵消"：每一步都有 $\varepsilon$ 量级的相对噪声，
> 经过 $N$ 次累加噪声方差涨为 $\mathcal{O}(N \varepsilon^2)$，
> 极端情况下相对误差可达 $\mathcal{O}(\sqrt{N}\,\varepsilon)$。

### 3.1 块结构对照

| 格式 | 元素宽度 | 块大小 | Scale 格式 | 每块字节数 |
|---|---|---|---|---|
| MXFP8 | 8 bit | 32 | E8M0 | 33 B（= 32 + 1） |

| MXINT8 | 8 bit | 32 | E8M0 | 33 B |
| **NVFP4** | 4 bit | **16** | **E4M3 (FP8)** + per-tensor FP32 | **≈ 4.5 bits/value** |

### 3.2 设计哲学一句话

- **FP64/FP32/FP16**：IEEE 754 标准，"通用计算"出身
- **BF16**：Google 为训练把 FP32 的尾数砍掉 16 bit，"宽范围派"
- **TF32**：NVIDIA 把 FP32 的尾数砍掉 13 bit，**只用于 Tensor Core 输入**，
  内存里仍是 FP32，"半截量化"
- **FP8**：OCP 标准，按"精度优先"(E4M3) vs "范围优先"(E5M2) 分两种
- **MXFP4/MXFP8**：OCP 微缩放，"块共享 scale 救动态范围"
- **NVFP4**：NVIDIA 把 MXFP4 块缩到 16、用 FP8 做 scale、加一层 FP32 全局 scale

## 4. 关键概念

### 4.1 范围 vs 精度：根本张力

- **范围**：能表示的数轴跨度，由 $2^{e_{\max}} - 2^{e_{\min}}$
- **精度**：相邻浮点数的相对距离，由 $2^{-m}$
- 训练时希望**范围**够大（梯度爆炸/消失），希望**精度**够高（避免抖动）
- **总位固定**时，e 和 m 只能二选一。这就是 BF16（8e+7m）vs FP16（5e+10m）的本质选择

### 4.2 累加器宽度：为什么 FP16/BF16/FP8 训练需要 FP32 累加

矩阵乘的内积和：$C_{ij} = \sum_k A_{ik} B_{kj}$，长度 $k$ 可达数千。

设乘积相对误差 $\le \varepsilon_{\text{mul}}$（FP16: ~2⁻¹¹，BF16: ~2⁻⁸，FP8: ~2⁻³ 量级），
假设**累加用同宽度**（最坏情形），误差方差累积为

$$
\text{rel.error} = \mathcal{O}(\sqrt{k}\,\varepsilon_{\text{mul}})
$$

- FP16 累加 + $k=4096$：相对误差 $\sim 2^{-5} \approx 3\%$——已不可忽略
- BF16 累加 + $k=4096$：相对误差 $\sim 2^{-2} = 25\%$——**完全不能用**
- FP8 E4M3 累加 + $k=4096$：相对误差 $\sim 2^{0} = 100\%$——崩溃

**结论**：FP16/BF16/FP8 训练必须用 **FP32 累加**。
这是 NVIDIA Tensor Core、AMD Matrix Core、Intel AMX 的硬性要求。
TF32 的累加也是 FP32，精度损失只是乘法那一步。

### 4.3 MX 共享 scale 的代价

共享 scale 把"块内动态范围"外包给一个 E8M0 量（$2^{-127} \sim 2^{+127}$），
但**块内的"相对精度"还是 $2^{-m}$**。

一个常见误解："MXFP4 比 FP16 好，因为 block scale 给了更多范围"。
实际上：

- 块内每个元素的表示精度**比 FP16 差得多**（E2M1 只能取 16 个离散值）
- MXFP4 的优势在于**自适应**——scale 能随块内数据分布迁移
- 没有 scale 时 E2M1 的 16 个值不足以拟合深度学习张量
- 有 scale 后，等效动态范围 ≈ FP16，但精度仍是 4-bit

这是 NVFP4 把块缩到 16 + 用 E4M3 做 scale 的动机：
更小的块 = 一个 outlier 影响更少；E4M3（带尾数）允许分数缩放，比 E8M0 的纯 2 幂次更精确。

### 4.4 一个有用的工程数值

对任何格式，**"相对误差 ≈ 2⁻ᵐ"** 都可以代回具体数字：

| 格式 | $2^{-m}$ 量级 | 直观含义 |
|---|---|---|
| FP64 | 1.1e-16 | 地球到太阳距离的厘米级 |
| FP32 | 1.2e-7 | 万分之一 |
| FP16 | 9.8e-4 | 千分之一 |
| BF16 | 7.8e-3 | 百分之一 |
| FP8 E4M3 | 1.25e-1 | 十分之一——**1/8 量化** |
| FP8 E5M2 | 2.5e-1 | 四分之一 |
| E2M1（无 scale） | 0.5 | 一半——这是为什么没 scale 的 FP4 几乎不可用 |

## 5. 选型决策表

| 场景 | 推荐格式 | 理由 |
|---|---|---|
| 通用科学计算 / 高精度求解 | FP64 | 唯一能稳定完成数值迭代的格式 |
| 通用 ML 训练（参数量 < 1B） | FP32 | 默认起点，无兼容性顾虑 |
| 大模型训练（参数量 ≥ 1B） | BF16 + FP32 master weight | 范围匹配 FP32，精度靠 master + 优化器补偿 |
| 已有 FP32 代码想用 Tensor Core | TF32 | 改动最小，仅作用于 matmul |
| LLM 预训练 / 大模型微调 | BF16 或 FP8 混合 | Hopper 之后 FP8 已成熟；E4M3 前向 + E5M2 反向 |
| 推理（≤ 13B LLM） | FP8 E4M3 | 精度足够，吞吐翻倍 |
| 推理（≥ 70B LLM / 长上下文） | NVFP4 / MXFP4 | 显存压到极限，KV cache 也可量化 |
| 边缘 / 微控制器推理 | INT8/INT4 | 算力优先，量化友好 |
