# bugs

> 代码诊断方法论：引导式问题集，从仓库整体到模块内部逐层深入。

---

## 概览

- **主题范围**：本目录不是 bug 记录簿，而是**诊断思维框架**——用一组精心设计的问题引导开发者去查看"值得看的地方"，而非打分。
- **核心隐喻**：诊断器而非裁判。不问"好不好"，而是问"哪里可能有问题"。
- **学习路径**：单篇笔记。

---

## 知识地图

### 代码诊断问题集
四层结构（仓库整体 → 模块 → 函数 → 实现），每层问题附"为什么问这个"——即诊断逻辑本身。

- **仓库整体层**：变更放大（"改一个需求平均动几个文件"）、依赖汇聚点（"哪些文件每次都动"）、循环依赖
- **模块层**：接口稳定性、单向依赖、抽象层次
- **函数层**：副作用、参数爆炸、命名清晰度
- **实现层**：魔法数、隐藏假设、注释与代码一致性
