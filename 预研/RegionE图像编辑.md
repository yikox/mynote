## 1. 问题动机

### 核心痛点
现有的 **Instruction-based Image Editing (IIE)** 模型存在严重的计算效率问题：

- 即使只修改图像的局部区域，模型也会对**整个图像**进行统一的去噪生成
- 编辑区域和非编辑区域在生成难度和计算冗余上**差异巨大**
- 现有方法忽略了这种差异，浪费大量算力在不需要变化的部分

### 关键观察
1. **非编辑区域的轨迹是直的**：可以在单步中推断多步去噪预测
2. **编辑区域的相邻时间步表现出强速度相似性**：可以利用缓存加速
3. 不同区域在生成难度上有本质区别，应该区分处理

## 2. 技术方案

### 2.1 三阶段框架

```
┌─────────────────────────────────────────────────────────────────┐
│                      RegionE 架构流程                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐       │
│  │  Stage 1     │    │  Stage 2     │    │  Stage 3     │       │
│  │  Early Stage │───▶│  Mid Stage   │───▶│  Late Stage  │       │
│  │              │    │              │    │              │       │
│  │ • 区域划分    │    │ • RIKVCache  │    │ • 全局信息融合│       │
│  │ • 识别编辑区  │    │ • AVDCache   │    │ • 质量恢复   │       │
│  │ • 识别非编辑区│    │ • 局部加速   │    │ • 完整去噪   │       │
│  └──────────────┘    └──────────────┘    └──────────────┘       │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 核心组件

#### Region-Instruction KV Cache (RIKVCache)
- **目的**：减少计算成本同时整合全局信息
- **机制**：在编辑区域生成过程中，丢弃未编辑区域的 token 和指令图像 token，但有效重新注入全局上下文
- **作用**：提高局部迭代生成的效率和质量

#### Adaptive Velocity Decay Cache (AVDCache)
- **目的**：利用相邻时间步的速度相似性加速局部去噪
- **机制**：精确建模去噪过程中速度衰减的模式
- **作用**：在编辑区域内部实现加速

### 2.3 区域划分策略

在早期去噪阶段，通过以下方式划分编辑/非编辑区域：

1. 获取最终估计结果与参考图像的差异
2. 基于差异图识别需要修改的区域
3. 非编辑区域使用跳跃式去噪（多步一跳）
4. 编辑区域使用缓存加速

## 5. 总结

RegionE 提出了一个**自适应、区域感知的生成框架**，核心贡献：

1. **问题洞察**：识别 IIE 任务中编辑/非编辑区域的本质差异
2. **技术方案**：通过 RIKVCache 和 AVDCache 实现高效区域处理
3. **实践价值**：即插即用，无损加速 SOTA 模型
