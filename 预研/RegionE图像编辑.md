# RegionE: Adaptive Region-Aware Generation for Efficient Image Editing

> 📌 **TL;DR** — 一个**训练免费**的图像编辑加速框架。**三阶段推理 + 区域感知生成**，将 IIE（Instruction-based Image Editing）速度提升 **2.06-2.57×**，PSNR 仅降 0.2-0.3 dB，**几乎无损**。
>
> **核心思想**：编辑任务中大部分像素其实没变 → 空间上把图分成"编辑区 / 非编辑区"分开算，用 KV Cache 注入被丢弃的全局信息 → 时间上用速度衰减跳步。

## 1. 问题与动机

**痛点**：IIE 模型对**整图统一去噪**，编辑区/非编辑区的计算冗余差异巨大。

**三个关键观察**：

1. **非编辑区**：去噪轨迹近似**直线** → 单步可外推多步
2. **编辑区**：相邻时间步速度**方向一致**、仅大小衰减 → 可跳步
3. **区域差异本质**：编辑/非编辑难度不同 → 应**分开处理**

## 2. 三阶段架构

| 阶段          | 步数范围        | 步数  | 关键操作                                |
| ----------- | ----------- | --- | ----------------------------------- |
| **STS** 稳态化 | t ∈ \[T, 16] | 6   | 标准去噪 + 缓存所有 KV                      |
| **RAGS** 区域感知生成 | t ∈ (16, 2]  | 14  | ARP + RIKVCache + AVDCache，step 16 强制刷新 |
| **SMS** 平滑 | t ∈ \[2, 0]   | 2   | 完整去噪（质量恢复）                          |
| **总计**      | —           | **22** | （原 28 步）                            |

**模型特定超参**：

| 模型              | η (ARP 分割) | δ (AVDCache 跳步) |
| --------------- | ---------- | -------------- |
| Step1X-Edit     | 0.88       | 0.02           |
| FLUX.1 Kontext  | 0.93       | 0.04           |
| Qwen-Image-Edit | 0.80       | 0.03           |

## 3. 三大核心组件

### 3.1 ARP — 区域划分

**做法**：在 STS 末（step 16）对最终图做单步欧拉估计 `X̂_0`，与原图 `X_ref` 比较相似度，按阈值 `η` 分成编辑区/非编辑区 mask。

```python
mask[i] = 1 if |X̂_0[i] - X_ref[i]| > η else 0
```

**为什么能分对**（核心原理）：

- 未编辑区轨迹是**直线**，用 t=16 处的速度 v 一步外推到 t=0，`X̂_0 ≈ X_ref` → 差异小 → mask=0
- 编辑区轨迹是**曲线**，同样的外推会飞离 `X_ref` → 差异大 → mask=1

> **关键洞察**：**预测误差即定位**——"编辑区"等价于"模型预测不准的区域"，自监督，无需 ground truth。

分割只发生一次（step 16），后续 RAGS 阶段复用。

### 3.2 RIKVCache — 空间冗余优化

**问题**：RAGS 步只跑编辑区 `X_E`，DiT 看不到非编辑区 `X_U` 和指令图 `X_I` → 局部生成缺上下文，会偏离。

**方案（三件套）**：

1. **只跑 x_e**：编辑区 token 数 `rN`（r 为编辑区占比），自然看不到 U/I
2. **t=16 强制刷新步建 cache**：全量 forward 一次，截取 U/I 对应的 K, V 存为 `K_C^U, V_C^U, K_C^I, V_C^I`
3. **attention 拼接注入**：

```text
A = softmax([Q_P, Q_E] · [K_P, K_E, K_C^U, K_C^I]ᵀ / √d) · [V_P, V_E, V_C^U, V_C^I]
```

**Cache 里的两类内容**：

| 缓存                 | 作用                                       |
| ------------------ | ---------------------------------------- |
| `K_C^U, V_C^U`     | 非编辑区：空间邻接上下文（边缘、纹理、阴影）                 |
| `K_C^I, V_C^I`     | 指令图：**全局风格锚点**（色彩、构图、风格）—— 不存可能丢失"原图是卡通风"这种全局约束 |

**位置编码适配**：依赖 RoPE（K 内嵌位置信息）→ 拼接时相对距离自动算出，无需对齐。**这是"训练免费"能 work 的隐藏前提**。

**Cache 刷新**：RAGS 内有强制刷新点（防 cache 漂移，论文未明说频率），SMS 阶段抛弃 cache 全量 forward。

**额外显存**：~0.5-1 GB（FLUX 量级），相对 24 GB 模型权重可接受。

### 3.3 AVDCache — 时间冗余优化

编辑区相邻步速度**方向一致、仅大小衰减**，可以跳步。

**核心公式**：


速度衰减：

$$‖v_ti‖ / ‖v_ti+1‖ = (1 - Δt_ti+1,ti) · γ_ti$$
跳步预测：v_tj   = v_ti · (1 - Δt_tj,ti) · γ_ti

**决策**：

```python
if |v_ti - v_ti+1| / |v_ti| < δ:
    skip_step()      # 速度变化小 → 跳步
else:
    compute_step()
```

**跳步示例**：t=T → t=0.9 → **跳** → t=0.8 → **跳** → t=0.6 → 算 → ...

### 3.4 AVDCache + RIKVCache 联合工作机制

**两者改的不是同一对象**：

- **RIKVCache**：改 `DiT.forward()` **内部**（attention 计算）
- **AVDCache**：改**外部** scheduler（决定是否调用 DiT）

**加速比相乘**：RIKVCache 让**单步变快**（token 少）+ AVDCache 让**总步数变少**（跳步）= 理论上 2.5× × 1.5× ≈ 3.75×，实测 2.54×（被"强制刷新点 + γ 反算"开销侵蚀）。

**关键耦合**：

- AVDCache **跳步**时 DiT 完全不跑 → RIKVCache 闲置
- AVDCache **算步**时 RIKVCache 才生效
- **强制刷新点 = AVDCache 不能跳**（RIKVCache 必须被使用）

**为什么 AVDCache 只能在 RAGS 跑**：

- **STS**：要建 cache，必须全量 forward；且刚启动没有 `v_{i-1}` 没法判跳
- **RAGS**：有 `v_{i-1}` 可比、有 cache 可用、有跳步价值
- **SMS**：接近收敛，速度衰减极快，跳过风险大；且要消除分块痕迹

## 4. 实验效果

### 4.1 整体性能

| 模型              | 延迟 (s)         | 加速比         | PSNR↑  | LPIPS↓ |
| --------------- | -------------- | ----------- | ------ | ------ |
| Step1X-Edit     | 18.93 → 7.45   | **2.54×**   | 29.43  | 0.053  |
| FLUX.1 Kontext  | 20.50 → 7.99   | **2.57×**   | 28.21  | 0.057  |
| Qwen-Image-Edit | 15.60 → 7.57   | **2.06×**   | 31.66  | 0.043  |

加速 2-2.5×，PSNR 损失 0.2-0.3 dB，**几乎无损**。

### 4.2 消融（Step1X-Edit）

| 变体              | PSNR            | 加速比    | 分析              |
| --------------- | --------------- | ------ | --------------- |
| **完整**          | 29.43           | 2.54×  | 基线              |
| w/o ARP         | 25.61 (−3.82)   | 2.79×  | ❌ 质量明显下降       |
| w/o RIKVCache   | 22.87 (−7.65)   | 2.73×  | ❌ 局部生成严重失真     |
| w/o AVDCache    | 29.41           | 1.65×  | ⚠️ 质量不变但加速比腰斩   |

**结论**：

- RIKVCache 决定**质量下限**（去掉 PSNR −7.65）
- ARP 保证**区域划分质量**（去掉 PSNR −3.82）
- AVDCache 决定**速度上限**（去掉加速比 2.54× → 1.65×）
- **三者缺一不可**：质、质、速各管一边

## 5. 与现有工作的区别

| 方法         | 缓存粒度     | 空间优化 | 时间优化 | 训练成本  |
| ---------- | -------- | ---- | ---- | ----- |
| Δ-DiT      | 时间步      | ❌    | ✅    | 需要训练  |
| FORA       | 时间步      | ❌    | ✅    | 训练免费  |
| **RegionE** | **patch 级** | **✅** | **✅** | **训练免费** |

**核心差异**：不修改权重（纯推理加速）+ patch 级区域感知 + KV 注入保证局部生成质量。

## 6. 总结

**RegionE = ARP + RIKVCache + AVDCache** 三件套：

| 组件            | 解决什么            | 怎么解决                |
| ------------- | --------------- | ------------------- |
| **ARP**       | 编辑/非编辑混合计算      | 区域划分，分开算            |
| **RIKVCache** | 局部生成缺全局上下文      | 缓存 U/I 的 KV，注入到 attention |
| **AVDCache**  | 每步都跑浪费          | 速度方向一致时跳步           |

**效果**：2-2.5× 加速，PSNR 损失 < 0.3 dB，**训练免费**，落地成本极低。

### 关键术语

| 术语                  | 含义                  |
| ------------------- | ------------------- |
| STS / RAGS / SMS    | 三阶段：稳态化 / 区域感知生成 / 平滑 |
| ARP                 | Adaptive Region Partition |
| RIKVCache           | Region-Instruction KV Cache |
| AVDCache            | Adaptive Velocity Decay Cache |
| IIE                 | Instruction-based Image Editing |
| η / δ               | ARP 分割阈值 / AVDCache 跳步决策阈值 |

| `K_C^U, V_C^U`      | 非编辑区的 cache KV      |
| `K_C^I, V_C^I`      | 指令图的 cache KV       |

### 思考与疑问

- **η/δ 怎么定**：三个模型用不同值，论文没明说，猜测 grid search 选 PSNR 最佳
- **强制刷新频率**：RAGS 内多久刷一次 cache 论文没说
- **位置编码兼容性**：依赖 RoPE，ALiBi 的老 DiT 不一定 work
- **Qwen 加速比偏低（2.06×）**：可能和其架构或 KV 合并策略有关
- **联合开销**："完整"加速比 2.54× 反而比"仅 ARP" 2.79× 低，强制刷新 + γ 反算的开销占比待量化
